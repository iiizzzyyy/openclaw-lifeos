# Facebook — Community, Discussion Questions

## Role
The group discussion. The "posting this because I keep seeing..." community moment.

## Audience
- Community group members
- Older demographic than other platforms
- People who engage in comments, not just likes

## DNA
- **Angle:** Community problem or shared experience
- **Hook:** "Posting this because..." or "Anyone else dealing with...?"
- **Tone:** Warm, inviting, peer-to-peer (not expert-to-student)
- **Format:** 1-3 paragraphs + optional image/link
- **Depth:** One discussion question, clearly stated

## Rules

### Must Have
- Relatable hook (shared experience)
- Context that invites "same here" responses
- Clear discussion question at end
- Warm, peer-level tone

### Never
- Expert lecturing (you're a peer, not a professor)
- Overly polished (feels like an ad)
- More than one discussion question
- Ignoring comments (reply to build community)

## Hook Formulas

### 1. The Shared Pain
> "Posting this because I keep seeing [problem] in [group/context]."

### 2. The Question
> "Anyone else dealing with [thing]? How are you handling it?"

### 3. The Confession
> "Admitting something: I struggled with [thing] this week."

### 4. The Observation
> "Noticing a pattern in [community]: [observation]. Thoughts?"

## Format Specs

- **Length:** 1-3 paragraphs (mobile-friendly)
- **Links:** Include at end (don't bury the post)
- **Images:** Optional, but personal > stock
- **Tone:** Warm, peer-level, inviting

## Structure

```
Hook (shared experience or observation)
↓
Context (why this matters to the group)
↓
Personal angle (your experience)
↓
Discussion Question (clear, open-ended)
↓
Optional: Link for those who want more
```

## Examples

### Bad (Reformat)
> "New blog post: AI Governance Best Practices. Read more: [link]"

### Good (Rethink)
> "Posting this because I keep seeing AI teams skip prompt versioning, and it's about to bite them when the EU AI Act hits.
>
> Sat in a compliance review last week. Room full of smart people. We could show every code change, full git history. Then someone asked 'show me the prompt version that generated this output.' Silence.
>
> We had Slack threads. Notion docs from months ago. Nothing structured.
>
> Anyone else dealing with this gap? How's your team handling prompt governance?
>
> (Wrote up the 3-step fix we're implementing if you want it: [link])"

## Repurposing Notes

| From Blog | Facebook Transformation |
|-----------|------------------------|
| Expert argument | Peer experience |
| "Here's the answer" | "How do you handle this?" |
| Comprehensive guide | One discussion point |
| Polished | Personal, warm |

- [ ] 1-3 paragraphs (mobile-friendly)?
- [ ] Link at end (not buried)?
- [ ] Willing to reply to comments?

## Platform-Specific Voice
Same tired engineer. But now they're in the community group, not on stage. Peer-to-peer, not expert-to-student. More "anyone else?" and less "here's what you should do."

## Posting Strategy
- **Frequency:** 2-3x per week (don't spam groups)
- **Best days:** Mon, Wed, Fri
- **Best times:** 8-10am, 7-9pm (when people check Facebook)
- **Group rules:** Check each group's self-promo rules before posting

## Engagement Tactics
- Reply to every comment in first 2 hours
- Ask follow-up questions ("What's your team doing?")
- Share other people's solutions (build community)
- Don't delete negative comments (unless spam/abuse)

## Group Types to Target
- AI/ML engineering groups
- Startup CTO communities
- Compliance/regulatory discussion groups
- Industry-specific tech groups (fintech, healthtech, etc.)
