# YouTube — SEO, Structured, 8-12 Minute Deep Dives

## Role
The tutorial. The "I'll actually teach you this" long-form content.

## Audience
- Search-driven learners
- People who want full context
- Builders implementing, not just browsing

## DNA
- **Angle:** How-to or deep explanation
- **Hook:** SEO title + thumbnail promise
- **Tone:** Teaching mode, structured, patient but direct
- **Format:** 8-12 minute video with chapters
- **Depth:** Comprehensive but actionable

## Rules

### Must Have
- SEO-optimized title (search intent + specific outcome)
- Thumbnail with clear promise (text + visual)
- Hook in first 15 seconds (what they'll learn)
- Chapters/timestamps in description
- CTA (subscribe, comment, related video)

### Never
- Bury the lesson (get to value fast)
- Vague titles ("AI Governance Explained")
- No structure (rambling without chapters)
- Over-promising (deliver what thumbnail promises)

## Title Formulas

### 1. The How-To
> "How to [achieve outcome] without [common pain]"

### 2. The Mistake
> "Stop [wrong approach]. Do [right approach] instead."

### 3. The Framework
> "[Number]-Step Framework for [specific outcome]"

### 4. The Reality Check
> "Why [common thing] fails (and what actually works)"

## Video Structure

```
0:00-0:15  Hook (what they'll learn, why it matters)
0:15-1:00  Context (the problem, stakes)
1:00-3:00  Core concept 1 (with example)
3:00-5:00  Core concept 2 (with example)
5:00-7:00  Core concept 3 (with example)
7:00-8:00  Implementation (how to start)
8:00-9:00  Summary + CTA
```

## Thumbnail Specs

- **Text:** 3-5 words max, large font
- **Contrast:** High (dark bg, light text or vice versa)
- **Face:** Optional (if using, show emotion relevant to topic)
- **Promise:** Clear outcome or problem being solved

## Example (AI Governance Topic)

**Title:** "AI Governance for Engineers: 3-Step Framework (EU AI Act Compliance)"

**Thumbnail:** "PROMPTS ≠ CODE" (split screen, red X on prompts, green check on code)

**Hook (0:00-0:15):**
> "If you can't trace which prompt generated that output, you aren't compliant with the EU AI Act. Today I'll show you the 3-step framework to fix that gap — same rigor as your code deployments."

**Chapters:**
- 0:00 The compliance gap
- 1:15 Why prompts aren't code (but need same treatment)
- 3:30 Step 1: Map touchpoints
- 5:45 Step 2: Implement versioning
- 7:00 Step 3: Build approval gates
- 8:30 Implementation checklist

## Repurposing Notes

| From Blog | YouTube Transformation |
|-----------|----------------------|
| Written argument | Spoken tutorial |
| Reader-paced | Editor-paced (chapters) |
| Skimmable | Linear with timestamps |
| Text examples | Visual demos/screen shares |

## Platform-Specific Voice
Same tired engineer. But now they're in teaching mode — whiteboard out, screen share ready, actually walking you through the thing. Patient but direct. No fluff, but more explanation than X/LinkedIn.

## Posting Strategy
- **Frequency:** 1-2x per week (quality > quantity)
- **Best days:** Tue-Thu, Sat-Sun
- **Best times:** 2-4pm CET (global audience overlap)
- **First 24 hours:** Reply to every comment (boosts algorithm)
