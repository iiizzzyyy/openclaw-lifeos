## 3. Execution Instructions

When given a blog post or topic, follow this exact sequence:

### Step 1: Niche Alignment Check
Before writing anything, verify the topic aligns with our niche:
- EU AI Act compliance
- LLM cost governance
- Prompt management/observability
- Production AI engineering challenges

**If the topic doesn't align:** Reject it and explain why. Don't force content that doesn't serve our audience.

### Step 2: Read Core Voice
Read `[[brand-voice]]` to calibrate personality. This is non-negotiable — every output must sound like Izzy (tired engineer, no fluff, real numbers).

### Step 3: Select Hook
Read `[[hooks]]` and select the best hook formula for this specific topic. Match hook type to platform:
- X/Threads: Contrarian, Proof, Replacement
- LinkedIn: Playbook, Behind-the-Scenes, Proof
- Instagram/TikTok: Discovery, Replacement, Contrarian
- YouTube: Playbook, Behind-the-Scenes
- Newsletter: Behind-the-Scenes, Discovery

### Step 4: Read Repurpose Chain
Read `[[repurpose]]` to understand:
- The order to write platforms (X first, always)
- What actually changes between platforms (angle, hook, tone, format, length, depth)
- The litmus test: "Would followers on all platforms be annoyed seeing the same thing?"

### Step 5: Write for X (First Platform)
Read `[[x]]` + `[[platform-tone]]` (X section).

Write the X version FIRST. X forces brevity and sharp hooks — if the core idea doesn't work here, it won't work anywhere.

Output: One tweet (280 chars) or thread (5-10 tweets, 280 each).

### Step 6: Adapt for LinkedIn (Second Platform)
Read `[[linkedin]]` + `[[platform-tone]]` (LinkedIn section).

**Do NOT just expand the X post.** Rethink the angle:

Output: Long-form post (1,300-2,000 chars) with personal narrative.

### Step 7: Continue Through Chain
For each remaining platform, repeat:
1. Read that platform's file (`[[instagram]]`, `[[tiktok]]`, etc.)
2. Read `[[platform-tone]]` for that platform
3. **Rethink the angle** — don't just reformat
4. Output one native post ready to publish

**Chain order:** X → LinkedIn → Instagram → TikTok → YouTube → Newsletter → Threads → Facebook

### Step 8: Apply Scheduling
Read `[[scheduling]]` and note:
- Best posting times for each platform
- Batch workflow (all 8 written in one session)
- Stagger publishing across the week per the calendar

## 4. Critical Rules

### Rule 1: Rethink, Don't Reformat
The output is **NOT** 10 copies of the same text reformatted for each platform. It's 10 pieces that each **THINK** about the topic differently.

Same topic, different:
- Angle (entry point)
- Hook (opener)
- Tone (energy)
- Format (thread vs carousel vs video)
- Length (280 chars → 2,000 words)
- Depth (surface insight → deep tutorial)

**Litmus test:** If someone followed you on ALL platforms, would they be annoyed seeing the same thing everywhere? If yes, you failed. Rethink.

### Rule 2: Voice Consistency
Every output must sound like Izzy:
- Direct, practical, allergic to fluff
- Real numbers > vague claims
- "Here's what actually works" energy
- Never: "In today's fast-paced world...", corporate buzzwords, "it's worth noting"

Read `[[brand-voice]]` before every writing session.

### Rule 3: Platform Native
Write for the platform's culture, not yours:
- X: Lowercase, punchy, contrarian
- LinkedIn: Professional but human, "I" stories
- Instagram: Visual-first, bold claims
- TikTok: Raw, energetic, 2-sec hook
- YouTube: Structured, educational, SEO-driven
- Newsletter: Personal, intimate, like a letter
- Threads: Relaxed, opinion-driven
- Facebook: Warm, community, discussion

Read the platform file + `[[platform-tone]]` before writing.

### Rule 4: Hook Discipline
80% of performance is decided in the first line. Never publish without testing 2-3 hook variations.

Track which hooks perform per platform. Update `[[hooks]]` every Friday with winners.

### Rule 5: Batch Workflow
Write all 8 platform versions in ONE session. Don't spread across days. Follow the chain order — each step feeds the next.

Review all 8 outputs together before publishing any.
